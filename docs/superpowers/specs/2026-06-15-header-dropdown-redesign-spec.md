# PitchLab 顶栏与个人管理中心重构规格说明书 (Spec)

## 1. 概述与背景
当前 PitchLab 后台管理系统存在顶栏（Header）元素过多、普通用户辅助功能（如 PRO 状态、邮箱、价格方案、退出登录等）与核心预测 Tab 按钮并排平铺、技术指标术语过于生硬的问题。

本设计规约旨在重塑顶栏与个人管理中心（PersonalHub），实现“极简且高端”的苹果毛玻璃卡片（Glassmorphic）视觉质感，彻底分离常规预测功能与管理设置选项。

---

## 2. 顶栏 (Header) 重构规划

### 2.1 界面元素清单与可见性
* **保留在顶栏的主元素**：
  * **⚽ Logo & 品牌名称**：`PitchLab` (点击后可重设 Tab 状态并返回冠军概率 Tab)。
  * **核心业务导航 Tabs**（普通用户只可见以下 8 项，管理员若有权限，其顶栏也不再平铺“个人管理”和“系统运维”）：
    * 冠军概率 (`title`)
    * 赛事预测 (`predictions`)
    * 价值发现 (`value`)
    * 历史回测 (`track`)
    * 联赛对比 (`leagues`)
    * 模型指数 (`league_model`)
    * 偏差反馈 (`feedback`)
    * 模拟交易 (`paper`)
  * **🌓 主题切换开关**：亮暗主题切换。
  * **👤 用户圆形头像 (Avatar)**：
    * 头像内部显示大写首字母，外圈在亮暗模式下均有平滑的白色边框。
    * **PRO 身份指示**：若是 PRO 用户，头像外围附加微妙的发光阴影特效，且头像边缘配有绿色的呼吸灯小圆点。
    * **交互**：点击后向下平滑动效展开一个浮动的毛玻璃卡片菜单（Dropdown Card）。

* **从顶栏中隐藏（收纳进下拉菜单）的次要元素**：
  * `个人管理` Tabs、`系统运维` Tabs、`PRO` 标识胶囊、邮箱文本、`价格方案` 按钮、`退出登录` 按钮、管理后台 `/admin` 跳转链接。

### 2.2 用户头像下拉菜单 (Dropdown Card) 交互设计
* **外观质感**：采用极佳的苹果毛玻璃卡片样式：`bg-white/70 backdrop-blur-xl border border-white/60 shadow-lg rounded-2xl`，支持平滑展开与折叠。
* **信息分区**：
  * **头部 (Profile Info)**：
    * 邮箱全称：`kakawah1122@gmail.com`
    * 订阅状态 Badge（`PRO 极速版` 或 `标准会员`）
    * 快捷账户统计：`测试筹码: 10,000`
  * **菜单项 (Actions)**：
    * ⚙️ 个人管理中心（点击后触发 Tab 状态更改为 `"system"`，下拉菜单收起）
    * 💎 升级与价格方案（点击链接至 `/pricing`）
    * 🛠️ 系统运维配置（仅 `isAdmin === true` 时显示，点击触发 Tab 状态更改为 `"system_admin"`）
    * 💻 数据管理后台（仅 `isAdmin === true` 时显示，Link 链接至独立页面的 `/admin`）
  * **底部 (Footer)**：
    * 分割线
    * **退出登录** (优雅的红色 hover，点击执行注销)

## 3. 个人管理中心 (Personal Hub) 重构规划

### 3.1 显示的功能板块 (去术语化普通用户体验)
* **HUD 头部卡片**：
  * 显示可用模拟测试筹码（Balance）余额、已激活筛选策略与订阅提醒的计数进度条。
* **左侧窄栏卡片 —— 数据接收终端**：
  * 极简 Telegram 推送绑定状态控制，包含绑定、解绑以及复制快捷绑定指令，带呼吸指示灯。
* **右侧宽栏卡片 —— 智能管理**：
  * **智能筛选策略配置 (Strategies)**：用户根据“胜率优势偏离阈值 (%)”、选择的联赛、是否开启 TG 同步推送来创建和删除跟单策略，系统后台自动追踪虚拟测试筹码战绩。
  * **实时赛事提醒订阅 (Alerts)**：按覆盖联赛、最小触发偏离阈值订阅开赛、终场及偏离警报。

### 3.2 彻底隐藏的底层组件 (收拢至“系统运维” Tab)
* ❌ **系统数据包同步面板 (DataSyncPanel)**：包含 Artifacts 同步文件明细表（普通用户不可见）。
* ❌ **模型计算管道状态监控 (SystemStatus)**：包含 Hermes 运行日志、Dixon-Coles 算法状态（普通用户不可见）。

## 4. 接口与多页面兼容设计
* 为了兼容 `/chat` 页面也使用了 `AuthBar` 组件的问题，`AuthBar` 应该接收两个可选 Props：
  * `currentTab?: string;`
  * `onTabChange?: (tab: string) => void;`
* 若未传递 `onTabChange`（例如在独立页面 `/chat` 页面上），点击下拉菜单中的“个人管理”时，直接执行 `window.location.href = "/?tab=system"` 实现跳转回主页并正确初始化该 Tab。

---

## 5. 验证标准
1. **导航栏纯净度**：顶栏不再有任何重合的账户邮箱文本、退出登录文本和个人管理 Tab 选项。
2. **头像下拉交互**：点击头像能在右下侧以 200ms 的平滑位移动效弹出毛玻璃卡片，且点击页面空白处或菜单项时自动折叠。
3. **隔离性**：标准用户在登录后无法在头像下拉中看到“系统运维”或“管理后台”；当未登录时，顶栏正常显示“账户登录”与“价格方案”。
4. **编译测试**：执行 `npm run build` 全站路由正常导出，没有任何 TypeScript 与静态 bailout 问题。
