设计方案草案 (Design Draft)

请仔细阅读以下各个设计模块，并在下方做出确认：

1. 路由与入口设计

建立新页面：apps/web/app/[sport]/league/[leagueId]/odds/page.tsx
当用户访问 /soccer/league/wc2026/odds 时，该页面会解析参数并渲染 Dashboard，同时传递 initialTab="odds" 以便默认打开“即时赔率”板块。

2. Dashboard 状态集成

修改 apps/web/components/Dashboard.tsx：
- 在 Tab 类型声明中增加 "odds"。
- 在 Tabs 切换按钮行，新增“即时赔率”标签按钮。
- 新增组件挂载逻辑：当 tab === "odds" 时，渲染 <OddsTableView leagueId={leagueId} />。

3. 前端组件 OddsTableView

新建 apps/web/components/OddsTableView.tsx，使用 PitchLab 渐变暗色系统与高对比毛玻璃表格容器：
- 完赛折叠：将比赛按日期分组。已完赛的比赛默认收起为折叠栏（如 "＋ 已完赛 · 6月12日 · 2场"），点击即可展开详情。
- 赔率与箭头：获取 Pinnacle 的 1X2 赔率。对比最新赔率快照与基准快照，计算箭头升降（升高为 ↑，降低为 ↓）。
- 投注链接与阶段：保留 "🔗" 投注外链和阶段分组超链接（如点击 "A组" 一键切回积分榜 Tab 的 A组区域）。

4. 健壮的动态数据拉取 (API)

数据获取支持双重容灾保障：
- Prisma 数据库：查询 Fixture 及关联的 OddsSnapshot 数据（仅加载 1X2 pinnacle 赔率）。
- 本地 JSON 容灾：如果数据库连接失败，自动 fallback 到 public/data/odds_snapshots.json 和 fixtures.json 文件以保证渲染可用性。
