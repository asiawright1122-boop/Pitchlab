# PitchLab 设计总览:功能 / 优势 / 怎么跑出来(复核版)

> 复核结论:**起步不用 Hermes(太重)。先轻量自建,借鉴其范式,按需再挂。**
> 时间:2026-05-29 · 这份是面向"动手"的设计收敛

---

## 0. 复核:为什么起步不用 Hermes

| | Hermes/OpenClaw 提供 | PitchLab 真正需要 |
|---|---|---|
| 本质 | 通用个人助理 Agent 平台 | 确定性量化研究流水线 |
| 自带 | 多聊天渠道/浏览器/RL/微调/语音/Canvas/全系统访问 | cron + 任务 + DB + 量化包 + 薄 LLM 封装 |
| LLM 用量 | 每步推理(成本高) | 仅 2 个窄节点(抽取/报告) |
| 风险 | 重、成本不可控($1.3M 教训)、绑定生态 | 轻、可控、可回测 |

> **决策修正(D12′)**:**Phase 0/1 完全自建轻量栈,不引入 Hermes。** 借鉴其"skill 文件 / MEMORY / cron"范式。等需要"自主多渠道/自学习技能"时,再把 Hermes 作为**可选外壳经 MCP 挂载**。

## 2. 功能清单(按模块)

### A. 概率引擎(核心,确定性)
- ⚙️ **比分分布引擎**:Dixon-Coles/双泊松 → **一次性派生** 1X2 / 亚盘 / 大小球 / BTTS / 比分
- ⚙️ **概率校准**:isotonic/Platt + Brier/可靠性曲线监控
- ⚙️ **ELO/攻防评分**:每场实时更新
- ⚙️ (后期)ML 集成:GBM/XGBoost

### B. 数据层(Adapter,mock 优先)
- 🔌 多源 Adapter:mock → football-data.co.uk(回测,含 Pinnacle)/ football-data.org(赛程)/ The Odds API(软盘)/ SportsGameOdds(锐盘)/ Understat(xG)
- 🧹 **名称标准化映射库**(球队/球员/联赛)+ 缺失处理 + 置信度标记
- 🧩 **特征库**:ELO / xG背离 / 伤停 / 战意 / 赛程疲劳 / 主场
- 🤖 **文本特征抽取**(伤停/阵容/舆情 → 结构化,便宜 LLM + 缓存)

### C. 策略与风控(给用户建议,不执行)
- 📐 **devig 去抽水** → 市场真实概率
- 🎯 **Value/EV 检测**:模型概率 vs 市场 → value 信号
- 💰 **Kelly 仓位建议**(给用户参考,不下注)

### D. 真相机器(护城河第一块砖)
- 🔬 **walk-forward 回测**:样本外 CLV / ROI / Brier
- 📊 按**联赛 / 市场 / 情境**分组的 CLV 曲线
- 🚦 Go/No-Go 闸门:找 CLV≥+2% 的 niche

### E. 自动化(轻量 cron,非重 agent)
- ⏰ cron:扫赛程 / 采集 / **队news 赛前高频监控** / 预测 / 结算 / 再训练
- ♻️ 幂等 + 可重放 + 快照(无未来信息泄漏)
- 🔁 滚动再训练(回路1)+ 校准(回路2)

### F. 产品/交付
- 🖥️ **Web 看板**:比赛列表 / 预测报告 / value 信号 / **公开 CLV 战绩曲线**
- 📣 多渠道推送:Telegram / Discord / Web(措辞合规:概率/分析,非荐彩)

### G. LLM 进化贡献(有护栏)
- 💡 假设/特征发现(读错题复盘 → 提新特征,交回测验证)
- 🔧 残差修正(有界+A/B+样本外 CLV 门槛+默认关闭)

## 3. 优势在哪(7 条)

1. **诚实可验证**:公开样本外 CLV/战绩曲线 —— 市面 99% 不敢(立身之本)
2. **量化严谨**:校准 + walk-forward + 防过拟合,不玩准确率幻觉
3. **一引擎全市场**:比分分布派生 1X2/亚盘/大小球,低成本广覆盖
4. **轻量低成本**:~$50–200/月,确定性流水线 token≈0,能活很久慢慢迭代
5. **复利数据壁垒**:映射库/校准参数/skill库/战绩曲线随时间增厚,后来者难追
6. **不下注 → 风险可控**:无本金/账号/牌照风险,最坏=没人用,不亏身家
7. **机构级产能**:小团队靠自动化跑出过去多人团队的研究产能

## 4. 怎么才能跑出来(收敛 §04)

### 4 个胜负手(按重要性)
1. ⚡ **真相机器先证伪**:2 周、$0,先回答"有没有 edge",再投入
2. 🎯 **聚焦一个低效 niche**(低级别联赛 + 亚盘/大小球)做到最强,别广撒网
3. 📣 **透明战绩做分发**:公开 CLV 曲线 = 信任 = 增长(对手不敢公开)
4. 🪶 **轻起步**:不背 Hermes 重包袱,KPI 只认 CLV/ROI/Brier

### 必死陷阱(避开=赢一半)
- 🪦 硬刚主流联赛收盘线 / 🪦 用准确率当KPI、回测调参过拟合 / 🪦 碰资金代理(已规避:不下注)/ 🪦 起步上重 agent 平台

### 现实预期
- 技术闭环 🟢 高 · 某 niche 找 +CLV 🟡 中 · C+B 产品有受众 🟡 中(取决于 niche+分发,非模型玄学)
- **下行可控**:即使无 edge,"诚实的分析工具(C)"依然成立

## 5. 落地顺序(Phase)

```text
Phase 0  真相机器(2周,$0)        回测框架+Dixon-Coles+CLV  → Go/No-Go
Phase 1  轻量骨架 + 数据模型        cron+DB+Adapter(mock)+Web列表
Phase 2  概率引擎 + 校准 + 策略层    devig/EV/Kelly + 校准
Phase 3  反馈进化 + 自动化          结算→CLV/再训练 + cron 跑通
Phase 4  产品化 C+B                公开战绩看板 + 多渠道推送
Phase 5  (可选)Hermes 外壳/自学习  按需挂载,不强求
```

并行:Phase 0(真相机器)与 Phase 1(骨架)可同时起步。
