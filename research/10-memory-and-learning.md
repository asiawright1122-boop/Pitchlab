# PitchLab #6:Agent 的记忆 与 自我学习(工程实现)

> 回应:"怎么实现 Agent 的记忆和自我学习?"
> 时间:2026-05-29 · 这份覆盖维度 #6(反馈进化工程细化)

---

## 0. 先分清两种"学习"(否则必做错)

| | 量化学习(学概率) | Agentic 学习(学流程) |
|---|---|---|
| 学什么 | 模型参数、校准、特征权重 | 分析流程、可复用技能、配置 |
| 怎么学 | 数据再训练 + 样本外验证 | 沉淀 skill + 记忆"什么有效" |
| 提升什么 | **准确率/校准/CLV** | **研究效率/复用/适应** |
| 主体 | 统计/ML 模型 | 编排器 + LLM(有护栏) |

> 红线(D8):**概率只能由量化模型学;LLM/Agent 学的是"怎么把研究做得更快更对",不是概率。**

## 1. 记忆架构(分 4 层)

```text
① 结构化记忆 (DB, 权威主体) ─────────────────────────────
   ├─ 实体映射库      team/player/league → canonical_id(清洗层累积)
   ├─ 模型仓库        model_version / 权重 / 超参 / 每联赛校准参数
   ├─ 特征有效性表    每特征的样本外贡献(ΔCLV/Δlogloss)随时间
   ├─ 预测快照        每次预测的特征+模型版本+赔率快照(CLV/重放必需)
   ├─ 结算与指标      result / CLV / ROI / Brier(按 比赛/联赛/市场)
   └─ 错题库          误判 + 归因标签(数据缺失/模型/方差)

② 技能记忆 (文件, 借鉴 Hermes) ──────────────────────────
   └─ skill/*.md|json  "如何分析 X 联赛/情境"的流程+配置(可复用/版本化/人读)

③ 语义记忆 (检索, 后期可选) ─────────────────────────────
   └─ 新闻/伤停文本的全文检索(先用 SQLite FTS5/关键词,够用再上向量)

④ 运行记忆 (run-log, 短期) ──────────────────────────────
   └─ 每次运行:输入快照/版本/产出(审计+重放+调试)
```

**设计要点:**
- 权威记忆在 **DB(结构化)**,不是塞进 LLM 上下文
- 借鉴 Hermes 但务实:**先 FTS5/关键词检索,不急上向量库**(省成本)
- 多租户:用户偏好/绑定/订阅也在 DB(区别于 Hermes 单用户的 MEMORY.md)

## 2. 自我学习:5 条回路(具体机制 + 读写哪块记忆)

### 回路1 · 滚动再训练(真正提升准确率)P0
- **ELO/攻防评分**:每场比赛结束**增量更新**
- **周期全量重训**:滚动窗口(如近 2–3 赛季),walk-forward
- 新权重写入**模型仓库(版本化)**,过样本外闸门才热更
- → 这是"准确率会变好"真正发生的地方

### 回路2 · 概率校准 P0
- 持续监控 **Brier/log-loss/可靠性曲线**(读结算指标)
- 周期 refit **isotonic/Platt**,每联赛独立
- 校准参数写入**模型仓库**

### 回路3 · 特征 ROI 闭环(系统自选因子)P1
- 每特征算**样本外边际贡献**(ablation/置换重要性 on CLV/logloss)
- 负贡献 → 自动降权/剔除;新特征先进**影子模式**
- 结果写入**特征有效性表** → 系统"学会"每联赛哪些因子重要

### 回路4 · LLM 假设生成 + Prompt/Skill A-B(Agentic 学习)P2
- LLM 读**错题库** → 提出新特征/新角度 → 自动进回测验证
- 成功的分析流程 → 沉淀为 **skill(技能记忆)** → 下次直接复用(Hermes 式复利)
- Prompt 版本化 A/B:以"抽取准确性 + 下游校准提升"为评判

### 回路5 · LLM 有护栏残差修正(最后,P3)
- 有界 ±X% + 版本化 + **样本外 CLV 门槛** + 默认关闭/自动熔断

## 3. 核心工程模式:Champion / Challenger + 影子模式 ⭐

让"自我学习"**安全且真实**的关键(像量化基金上线新策略):

```text
当前线上 = Champion(冠军模型/特征集/Prompt)
任何"学到的新东西"(新权重/新特征/新Prompt/LLM修正) = Challenger(挑战者)
        │
   先在【影子模式】跑:与 Champion 并行预测,不影响线上,只记录
        │
   样本外评测:Challenger 的 CLV/Brier 是否**显著优于** Champion?
        │
 ┌──────┴──────┐
 是(过闸门)      否
 │              │
晋升为 Champion   丢弃/继续观察
(可一键回滚)
```

**配套护栏(防过拟合/退化):**
- ✅ **一切样本外**(walk-forward),绝不在回测集调参到天花板
- ✅ **版本化 + 回滚**:每次变更可追溯、可撤销
- ✅ **漂移检测**:线上实时 CLV 偏离回测 CLV → 报警(edge 衰减/数据问题)
- ✅ **多重检验惩罚**:试了 N 个策略要对显著性打折(防"试 100 个总有 1 个看着赚")
- ✅ **充分样本**:CLV 闸门需 ~200–500 注;盈亏判断 2000+ 注

---

## 4. 一句话总结实现

> **记忆 = DB 结构化主体(映射/模型/指标/快照/错题)+ 文件式可复用 skill + 运行日志。
> 自我学习 = 量化模型滚动再训练&校准(学概率)+ 特征 ROI 自选 + LLM 假设/技能复利(学流程),
> 全部用 Champion/Challenger + 影子模式 + 样本外闸门 保证"只升级真正变好的东西"。**

诚实预期:自学习让系统**更校准、更少噪声、能适应漂移**,但不会"学到"打穿有效市场——它让你**持续诚实、持续适应**,而非全知。
