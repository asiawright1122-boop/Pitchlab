## 0. 最重要的判断:LLM 不应该是"概率引擎" ⚠️

> **足球预测的核心概率,不能交给大模型生成。LLM 负责"理解数据"和"表达分析",不负责"算概率"。**

为什么?
- 市场(Pinnacle 收盘线)校准误差 < 1%,是地表最强的真实概率估计
- LLM 给概率 = 主观、不可校准、不可回测、每次还不一样 → **无法验证 CLV,无法进化**
- 48% 的结果,本质就是"让 LLM 当概率引擎"的必然天花板

**正确分工:**

| 角色 | 由谁做 | 产出 |
|---|---|---|
| **概率引擎(核心)** | **量化模型**(Dixon-Coles / 双泊松 / 梯度提升 + ELO + xG) | 可校准、可回测的胜平负/比分概率 |
| **数据理解** | LLM | 把新闻/伤停/阵容/舆情等**非结构化文本**抽成结构化特征 |
| **表达层** | LLM | 生成人类可读的分析报告 |
| **(可选)微调** | LLM(带护栏) | 在量化概率基础上做有限修正,且修正必须可回测、默认关闭 |

> 这一条直接改写架构:**第四层"预测"= 量化概率引擎 + LLM 解释器**,而不是"调一次大模型出答案"。

## 1. 特征工程优先级(按对 edge / CLV 的 ROI 排序)

经验 + 文献共识,**高 ROI 在前**:

| 排名 | 特征 | 价值 | 成本 | 数据源 | 说明 |
|---|---|---|---|---|---|
| 🥇 1 | **市场赔率本身(开盘/变盘)** | 极高 | 低 | The Odds API / SportsGameOdds | 市场已编码一切,是最强单一特征 + 基准;变盘方向=聪明钱信号 |
| 🥈 2 | **球队实力评分(ELO / Dixon-Coles 攻防 / xG 评级)** | 高 | 低(自算) | 自算 + 历史结果 | 性价比之王,稳定可回测 |
| 🥉 3 | **xG / 底层表现(运气校正)** | 高 | 中 | Understat | 比"胜负战绩"准,识别"赢球但表现差/输球但表现好" |
| 4 | **核心球员伤停 / 首发(且市场未充分定价时)** | 高(有时效) | 中 | API/Transfermarkt + LLM 抽取新闻 | **真正的 edge 多来自这里**——抢在市场调整前拿到队news |
| 5 | **赛程疲劳 / 战意(轮换、欧战后遗症、死亡之组、保级争冠)** | 中 | 中 | 赛程推算 + LLM | 特定情境有真实 edge,难量化 |
| 6 | 主场优势 | 中(已被定价) | 低 | 自算 | 边际信息少,但仍是基线项 |
| 7 | 历史交锋(H2H) | 低 | 低 | 历史 | **被高估**;样本小、风格克制信号弱,易噪声 |
| 8 | 近期连胜/连败"手感" | 低 | 低 | 历史 | **被高估**;大部分已被评分/xG 吸收,生数据很噪 |
| 9 | 天气 / 裁判 | 很低 | 中 | 杂源 | 小众,边际 |

### 关键洞察
- **edge 不在"更多特征",在"市场没充分定价的信息"**:首选 **#4 队news 时效** 和 **#3 xG 与结果的背离**
- H2H、近期手感这类"看起来很专业"的特征,**ROI 很低**,别投入太多
- **市场赔率(#1)既是最强特征,也是你的对手**——你的模型概率必须独立于市场算出来,再和市场比,才知道有没有 edge

## 2. 选哪个市场下手?(影响难度)

抽水越低,需要的 edge 越小,越容易盈利:

| 市场 | 抽水 | 出口数 | 建议 |
|---|---|---|---|
| **亚盘(Asian Handicap)** | 2–5% | 2 | ⭐ 推荐先做:抽水最低、二选一、Dixon-Coles 天然适配 |
| **大小球(O/U)** | 3–6% | 2 | ⭐ 推荐:双泊松直接输出进球分布 |
| 1X2 胜平负 | 3–8% | 3 | 抽水高、含平局难度大,作为展示但不作主战场 |
| 正确比分 | 15–30% | 多 | ❌ 抽水巨高,别碰 |

> **建议主战场:亚盘 + 大小球**(双泊松/Dixon-Coles 天然输出,抽水低)。1X2 用于内容展示。

## 3. Agent 架构设计

### 3.1 总体:量化引擎 + LLM 工作流,而非"一次问答"

```text
┌─────────────────────────────────────────────────────────────┐
│                      PitchLab Agent (Hermes 式)               │
│                                                              │
│  [调度 Scheduler/cron] 每日扫描未来 72h 比赛                    │
│        │                                                     │
│        ▼                                                     │
│  L1 采集 Ingest ──► L2 清洗标准化 ──► L3 特征加工              │
│  (Adapter 多源)     (名称/时区/缺失)   (ELO/xG/伤停/战意)       │
│        │                                                     │
│        ▼                                                     │
│  L4 概率引擎 (Dixon-Coles/双泊松/GBM)  ◄── 模型权重(可热更)    │
│        │  + 校准 (isotonic/Platt)                            │
│        ▼                                                     │
│  L4.5 LLM 解释器(把概率+特征→可读分析,+ 抽取文本特征回流 L3)  │
│        │                                                     │
│        ▼                                                     │
│  L5 策略 Strategy: devig 市场 → 比较 → +EV/value-bet 筛选     │
│        ▼                                                     │
│  L6 资金管理 Staking: Kelly 定注量(纸面/实盘)                 │
│        ▼                                                     │
│  L7 反馈进化 Feedback: 赛果回填 → CLV/ROI/Brier → 归因 → 再训练 │
│        │                                                     │
│        └──────────────► 模型权重热更 / 特征增删 / Prompt 迭代   │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 Agent 是"工作流式"而非"问答式"
每场比赛是一条**确定性流水线**(每步可单测、可重放、可审计):
`采集 → 清洗 → 特征 → 概率 → 解释 → devig → EV 筛选 → 注量 → 落库 → (赛后)结算 → 归因`

LLM 只出现在两个**有界**节点:
- **文本特征抽取**(伤停/舆情/阵容新闻 → 结构化字段),输出 schema 受约束
- **报告生成**(概率 + 特征 → 自然语言),不参与算概率

### 3.3 关键工程原则
- **可重放(replayable)**:任意比赛、任意时间点的输入快照都能复现预测(回测必需)
- **无未来信息泄漏(no look-ahead)**:特征只能用"预测时点前"已知的数据(回测正确性的生命线)
- **快照存储**:每次预测存下当时的特征 + 模型版本 + 市场赔率,赛后才能算 CLV
- **幂等任务**:cron 重跑不产生脏数据

## 4. 自我学习 / 进化机制(核心:进化的是"量化模型",不是 LLM)

> 误区:"让大模型从结果里学习"。现实:LLM 无法靠喂结果自动变准,**真正进化的是量化概率模型 + 校准 + 特征集 + Prompt**。

### 四条进化回路

#### 回路 1:模型再训练(Online / Rolling)
- 每轮比赛结束 → 结果入库 → **滚动窗口重训**(如最近 2–3 个赛季)
- ELO/球队评分**每场实时更新**
- 输出:新模型权重 → 热更到 L4

#### 回路 2:概率校准(Calibration)
- 持续监控 **Brier score / log-loss / 可靠性曲线**
- 用 **isotonic / Platt** 重新校准,让"模型说 60%"真的 60% 命中
- 校准好 = CLV 才可信

#### 回路 3:特征 ROI 闭环(自动增删特征)
- 跟踪每个特征对**样本外 CLV/log-loss 的边际贡献**
- 贡献为负/不显著 → 自动降权或剔除(对抗 H2H/手感这类噪声特征)
- 新特征先进"影子模式"(shadow)跑一段,证明有用再启用

#### 回路 4:Prompt / Agent A-B 迭代
- 对 LLM 的抽取/解释 Prompt 做版本化 + A/B
- 用"抽取准确性"和"是否提升下游校准"做评判,保留更优版本

### 防过拟合(进化系统最大的坑)⚠️
- **Walk-forward / 时间序列交叉验证**,绝不用未来数据训练
- 只看**样本外(out-of-sample)** CLV,严禁用回测集调参到天花板
- **正则化** + 控制特征数(特征越多越容易过拟合赔率噪声)
- 警惕**幸存者偏差**和**多重检验**(试 100 个策略总有 1 个"看起来赚")
- 红线仍是 **样本外 CLV ≥ +2%**;盈亏要 2000+ 注才可信

| 层 | 名称 | 核心职责 | LLM 参与? |
|---|---|---|---|
| L1 | 原始采集 | Adapter 多源拉数 | 否 |

## 7. LLM 到底能不能"学习进化"?(精确回答你的追问)

> 结论:**能,但要分清"哪种进化"。LLM 不能学会"更准的概率",但能在 4 个真实环节驱动系统进化。**

### LLM 不能做的(别期待)
- ❌ 靠喂结果"学会"输出更准的概率(无梯度、上下文学习有限、无法稳定跑赢市场)
- ❌ 自由覆盖量化模型的概率(不可校准 → 破坏 CLV)

### LLM 能真实贡献进化的 4 个环节 ✅
1. **假设/特征发现器(最有价值)**:LLM 读"错题复盘"(这些比赛预测错了,共性是什么)→ **提出新特征/新角度** → 交给量化回测验证。人/量化负责证伪,LLM 负责发散。
2. **文本特征抽取器(持续变强)**:把伤停/阵容/舆情/教练访谈等**非结构化文本**抽得更准更全 → 喂给量化模型 → 模型变准。LLM 这块靠 Prompt 迭代进化。
3. **错误归因器**:赛后判断"错在数据缺失 / 模型 / 还是纯方差" → 路由到正确的修复动作。
4. **有护栏的残差修正器(你要保留的那层)**:在量化概率之上做**有界、小幅**修正。

### 残差修正层的护栏(满足你"保留 LLM 修正"的诉求)
- **有界**:修正幅度 cap(如概率最多 ±X%),不能推翻量化结论
- **版本化 + A/B**:每个 Prompt/策略版本独立追踪
- **样本外 CLV 门槛**:必须在 out-of-sample 上证明 **提升 CLV** 才启用
- **默认保守 / 自动熔断**:无证据提升就自动关闭,绝不"信任 LLM 直觉"

> 即:**"进化的主体仍是量化模型 + 回测闭环";LLM 是这个闭环里的"发散引擎 + 文本传感器 + 有护栏的微调旋钮",而不是学习概率的主体。**

## 8. 一个引擎产出全部市场(回应"三者都要")

用 **双泊松 / Dixon-Coles** 估出"主客队进球数的联合分布(比分矩阵)"后,**一次性**可导出所有市场的概率:
- **1X2**:对比分矩阵按 主胜/平/客胜 求和
- **亚盘 AH**:按让球线求和
- **大小球 O/U**:按总进球阈值求和
- **BTTS / 正确比分**:同一矩阵直接读

> 含义:"三者都要"**几乎不增加成本**——核心只需做好一个"比分分布引擎",所有市场是它的派生。这也让 1X2 作主战场变得合理。

## 9. 自我进化回路:最终优先级(我定)

按"先让 CLV 可信 → 再让系统自动变强 → 最后上 LLM 高风险环节"排序:

| 优先级 | 回路 | 为什么这个顺序 |
|---|---|---|
| **P0** | 回测框架 + **回路2 校准**(Brier/可靠性曲线/isotonic) | CLV 可信的**前提**,没校准一切免谈 |
| **P0** | **回路1 滚动再训练**(ELO 每场更新 + 周期重训) | 基础进化动力,几乎零额外成本 |
| **P1** | **回路3 特征 ROI 闭环**(样本外贡献,自动剔噪声) | 中期,持续提纯特征 |
| **P2** | **回路4 Prompt A/B + LLM 假设生成/文本抽取** | 中后期,LLM 的安全进化贡献 |
| **P3** | **回路5 LLM 有护栏残差修正** | 最后、最高风险,必须样本外证明 +CLV 才启用 |

> 一句话:**先把"会算账"做对(P0),再让它"自己变聪明"(P1),最后才让 LLM"出主意+微调"(P2/P3)。**
