# PitchLab 重新定位:体育垂直的"自主 Agent + 量化研究"产品

> 关键澄清:**作者不做博彩,只做分析(类比量化交易的研究/信号端)。**
> PitchLab = 把 Hermes Agent / OpenClaw 这类**自主 Agent 范式** + **量化方法论**,应用到足球垂直。
> 时间:2026-05-29 · 这份文档基于澄清后的定位,对 02/04 做修正与深化。

---

## 0. 定位重述(一句话)

> **PitchLab 是"足球版量化研究台 + 自主 Agent":你做的是 alpha 研究、概率建模、价值信号、回测验证,以及把这一切自动化的 Agent。你不下注——下注是用户的事,就像量化研究员产出信号,但不亲自拿钱去赌。**

这把项目从"博彩工具"彻底拉到"**数据分析 / 量化研究 / Agent 产品**",和你说的 Hermes/OpenClaw 用在体育领域完全对齐。

## 1. 你不下注 → 整个风险结构改变(关键)

| 维度 | 旧理解(含自营下注) | 新定位(纯分析/研究) |
|---|---|---|
| 法律/合规 | 🔴 高(博彩、资金代理) | 🟢 低(数据分析产品,不碰资金) |
| 资金风险 | 🔴 本金可能血本无归 | 🟢 无本金风险 |
| 账号风险 | 🔴 软盘封号、锐盘限额 | 🟢 不存在(你不开投注账户) |
| 下行最坏情况 | 血本无归 | "产品没人用"(可控) |
| 成功取决于 | 能否赢庄家 | **信号质量 + Agent 产品力 + 分发** |

> **结论:模型 A(自营投注)彻底退场,降级为"内部 paper-trading(纸面跟单)验证信号质量",不投真金、不带客下注。** 这让项目下行风险从"亏身家"变成"做了个没人用的产品",**风险等级大幅下降**。

## 2. 把足球当作一个"市场":量化交易类比映射

这是理解整个系统最有用的心智模型——**足球博彩市场 ≈ 一个金融市场,你做 sell-side/buy-side 研究**:

| 量化交易概念 | PitchLab 对应 |
|---|---|
| 资产价格 | 博彩赔率 |
| 资产"公允价值" | devig 去抽水后的真实概率 |
| Alpha(超额信号) | 模型概率 vs 市场概率的偏离(value) |
| 基准(benchmark) | Pinnacle 收盘线(地表最准) |
| 信息比率 IR / 夏普 | **CLV 的稳定性 / 显著性** |
| 交易成本 / 滑点 | 庄家抽水(margin) |
| 因子(factor) | 特征(ELO/xG/伤停/战意…) |
| 因子拥挤 / 衰减 | edge 被市场吸收后失效 |
| 回测 / 样本外 | walk-forward,样本外 CLV |
| 风险管理 / 仓位 | 信号置信度 + Kelly **建议**(给用户,不自营) |
| 交易执行 | **用户自己做,你不碰** |
| 研究产出 | 概率、value 信号、分析报告、回测曲线 |

> 你的产品 = **"足球量化研究台"**:产出信号与分析,用 CLV 当客观质量标尺,**像 Bloomberg/quant 终端,不像庄家也不像赌客**。

## 3. PitchLab 与 Hermes / OpenClaw 的关系

两者都是**通用的自主 Agent 运行时**:自托管、持久记忆、**自学技能(agentskills.io)**、cron 调度、多渠道交付(Telegram/Discord/Slack/Web)、工具编排(浏览器/代码/MCP)。

PitchLab 可以走两条路:

| 路线 | 做法 | 优劣 |
|---|---|---|
| **A. 构建在其之上** | 把 PitchLab 做成 Hermes/OpenClaw 的**足球量化 skill 包 + workflow** | 复用现成 agent 能力(cron/记忆/多渠道/浏览器);快;但绑定其生态、成本不可控(见 OpenClaw $1.3M 账单警示) |
| **B. 自建轻量垂直 runtime** | 自己实现"cron + skills + memory + 多渠道"的精简版,只为足球量化服务 | 可控、成本低、可商业化为独立产品;但要自己造部分轮子 |
| **C. 混合** | 核心量化流水线自建(可控/可回测),把"多渠道交付/浏览器采集"等借 Hermes/OpenClaw 或其 MCP | 平衡 |

> 我的倾向:**C 混合**——量化与回测必须自建(可控、可审计、可回测是生命线);采集/推送/记忆等"Agent 外围能力"可借鉴或对接 Hermes/OpenClaw 范式,避免重复造轮子,也控制成本。

## 4. Agent 自主 & 自学习设计(借鉴 Hermes 的 skill loop,但守住量化内核)

Hermes 的精髓:**完成复杂任务 → 沉淀为可复用 skill → 下次直接调用 → 越跑越快越强**。把它用对地方:

| Hermes/OpenClaw 能力 | PitchLab 怎么用 | 注意 |
|---|---|---|
| **自学技能 Skill** | 把"如何分析 X 联赛 / X 情境"的**流程**沉淀为 skill(清洗映射、特征配方、校准参数) | 沉淀的是**流程与配置**,不是"学概率" |
| **持久记忆 Memory** | 记住每联赛名称映射、特征有效性、历史校准、已知坑 | 复用 D2 决策记录思路 |
| **cron 调度** | 每日扫未来 72h → 采集 → 建模 → 回测刷新 → 推送研究 | 幂等、可重放 |
| **多渠道交付** | Telegram/Discord/Web 推送**分析与概率**(非荐彩话术) | 合规:措辞用"概率/分析",不用"稳赚/必中" |
| **工具编排(浏览器/MCP)** | 采集 Understat/Transfermarkt(反爬时用浏览器自动化) | 控制 LLM token 成本 |

> 守住红线(呼应 D8):**Agent 的"自学习"是流程/技能层面的复用与提效;概率核心仍是可校准、可回测的量化模型。LLM 不学概率,Agent 学的是"怎么把研究流程做得更快更好"。**

## 5. 重新定义产品形态(C+B,A 退为内部验证)

| 形态 | 描述 | 收入 | 合规 |
|---|---|---|---|
| 🟢 **C 量化研究台 / 工具** | 概率、value/EV、devig、CLV、赔率比较、回测可视化;自主 Agent 跑数据 | 工具订阅 / API | 低风险(分析工具) |
| 🟡 **B 研究/信号订阅** | Agent 自动产出研究报告 + **透明 CLV 战绩**背书 | 内容/信号订阅、社区 | 中(避免保证盈利话术) |
| 🔵 **Agent 能力本身** | 作为"足球量化垂直 Agent"或 Hermes/OpenClaw 的 skill 包 | 技术/授权/demo | 低 |
| 🔴 **A 自营下注** | **退场** → 仅内部 paper-trading 验证信号质量 | 无(不投真金) | — |

> 护城河 = **量化严谨(可验证 CLV)× Agent 自动化(自主、自学习、多渠道)× 垂直深度(联赛 niche)× 透明战绩(分发信任)**。这正是"小团队拥有过去机构级研究产能"的体现。

## 6. 这对"怎么跑出来"的修正(深化 04)

§04 的作战策略**依然成立**,但有三处强化:

1. **下行风险可控**:不押身家,最坏是"工具没人用"。→ **可以更大胆地做产品、做分发**,不必等 edge 证明才动手
2. **CLV 仍是核心,但角色变了**:从"我赚钱的前提"→"**我研究/信号质量的客观招牌**"。即使某联赛 CLV 只是微正,作为"透明、可验证的研究产品"仍有卖点(市面 tipster 99% 不敢公开 CLV)
3. **Agent 自动化本身就是产品力**:Hermes/OpenClaw 证明了"自主、自学习、多渠道 agent"有巨大需求。把它跑通在足球量化垂直,**agent 工程本身 = 差异化**,不全押在"模型多准"
