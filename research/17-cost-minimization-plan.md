## 2. 极简技术架构设计 (The $0 Stack Architecture)

为了实现 $0/月 的运行成本，我们需要将传统的“长驻服务器 (Long-running Server)”架构改造为**“无服务器事件驱动 (Serverless & Event-Driven)”**架构：

```
                              ┌──────────────────────────────┐
                              │  GitHub Actions (定时执行)   │
                              └──────────────┬───────────────┘
                                             │ (1. 抓取赔率/赛程，写入DB)
                                             ▼
┌──────────────────┐           ┌──────────────────────────────┐
│  Telegram Client │ ◄────────►│  Vercel Serverless (TS)      │
└──────────────────┘ (2. Webhook│  (负责接收 TG 消息/卡片交互)  │
                      交互)    └──────────────┬───────────────┘
                                             │ (3. 读写数据 / 认证)
                                             ▼
                              ┌──────────────────────────────┐
                              │  Supabase Free (Postgres)    │
                              └──────────────────────────────┘
```

### 2.1 数据流运行机制
1.  **数据抓取（GitHub Actions 工作流）**：
    *   **每日 08:00 UTC**：启动 GitHub Action 脚本，调用 `football-data.org` 获取今日比赛列表，存入 Supabase。
    *   **赛前 T-6 小时**：启动 Action 脚本，调用 `The Odds API` 免费额度拉取当前盘口赔率并写入 DB。
2.  **消息交互（Telegram Webhook ➔ Vercel Serverless）**：
    *   将 TG Bot 的 Webhook 绑定 to Vercel 部署的 Serverless 接口。
    *   当用户在 TG 群里点击“投注卡片”时，Vercel 接收到事件，直接调用 Supabase 扣除虚拟代币并写入投注记录。整个过程不到 100 毫秒，且没有常驻服务器开销。
3.  **批处理结算（每周两次数据刷库）**：
    *   每周三、周日晚上，GitHub Action 触发结算脚本，下载 `football-data.co.uk` 免费 CSV，对比收盘赔率，批量更新 Supabase 中的投注状态，并计算用户的 CLV 得分。

## 3. 前期冷启动的“省钱”运营法则

除了技术成本，运营开销也必须降到最低：

1.  **拒绝精细化 UI，拥抱 TG 按钮卡片**：
    *   冷启动阶段**不要开发复杂的 Telegram Mini App (Web) 页面**。直接使用 Telegram Bot 的原生 Reply Markup 按钮和 Inline Keyboards（行内卡片）。这不仅开发速度极快（1个工作日即可跑通），而且完全没有前端托管和适配成本。
2.  **善用免费的社区流量**：
    *   不要在 Facebook/Google 投放任何应用安装广告。
    *   **获客动作**：创始人每天在 Twitter/Reddit 上搜寻吐槽“我的曼联投注单被收盘绝杀了”或者“庄家又吃水”的球迷，私信或回复他们：“这里有一个免费工具，可以自动计算你的投注单 CLV，帮你审计你的真实眼光，要不要进 Telegram 试一下？”
3.  **金币体系极简版**：
    *   第一期不急于接入 Adsgram 视频广告。只需在 Bot 里写死：*“初始赠送 1,000 金币。输光后，拉 1 个好友进群并 @Bot，即可自动补给 500 金币。”* 强行用社交裂变代替广告变现，先验证 K-factor 是否大于 1。
